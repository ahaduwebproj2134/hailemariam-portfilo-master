Why are there so many different programming languages out there?


Since programmers are human beings, they may have different test in choosing between coding languages for style, performance, and many other reasons. Apart from this, some note that different programming languages are best suited for different functions. In that they reiterate Ruby and Java Script are best for building web sites while Java and C++ are well suited for financial trading or game development, so forth. Besides, the commercialization of computers may also be another factor for the proliferation of programming languages. Companies have their own profit ends, and hence it can rightly be argued that introducing new languages gives them a competitive advantage.

What is the Role of Software Developer at the Implementation Phase?


I want to discuss about the role of the developer in implementation phase of software development life cycle (SDLC). Implementation comes after the software development process have passed through the sages of requirement specification, system analysis and system design. At this stage the developer performs functions of coding and self-testing. The designed program has to be changed to an accrual program. Hence, the developer will start building the entire system by writing code using the chosen programming language based on design documents and outlined specifications. In here, the developer has to follow any coding guidelines set by the organization. By executing a self-testing, the developer, a verifies whether or not the coding brings the expected results as defined. Self-testing is different from testing by done QA engineers at the fifth stage of SDLC. While the goal of self-testing by the developer is to ensure the new feature is successfully integrated with the system, QA specialists conduct overall testing to identify errors from various aspects. Self-testing, as a process, involves debugging whereby the developer identify, analyse and fixes errors to produce a bug free software. The developer my use debugging tools such as code-based tracers, profilers, and interpreters

What is Object-oriented programming? And, why do we use it?

Object-oriented programming is a style of programming that designs coding into divisions of objects that communicate to each other. Each object has its own distinctive properties with the ability to be accessed and modified. Peter Wegner on his article entitled ‚ÄúConcepts and paradigms of object-oriented programming‚Äù, published on ACM SIGPLAN OOPS, mentioned the four the goals/importance of object-oriented programming into four. The first one is management of software components. He stated that Object-oriented programming reintroduces systematic techniques for managing complex software components. Secondly, he noted that object-oriented libraries are classes from which objects may be created. Since such classes offer unrestricted availability, it increases their reusability, and provides uniformity. Third, he maintains that object-oriented programming promotes the inter-application reusability for system classes and development reusability which is essential to have a cost-effective software productivity. Lastly, Peter retreats that Object-oriented programming provides a framework for the seamless management of large software/application system. In that it enables the effective management of large programs and saves the time it takes to code a program.
